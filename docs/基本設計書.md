# UniConnDBBridge ― 最終仕様書 (2025-06-26版)

すべての合意内容を **重複なく** 1 本に統合した決定版です。
このドキュメントだけでリポジトリを用意し、開発・公開まで進められます。

---

## 1. 概要

| 項目      | 内容                                                                |
| ------- | ----------------------------------------------------------------- |
| ライブラリ名  | **UniConnDBBridge**                                               |
| 目的      | SQLAlchemy が扱えるあらゆる RDBMS・認証方式を透過化し、同期／非同期 API を一本化               |
| 主な特徴    | ドライバ自動選定、接続情報自動判定、Poetry パッケージ、MkDocs リファレンス、GitHub Actions CI/CD |
| ライセンス   | MIT                                                               |
| バージョニング | CalVer `YY.MM.PATCH`（例 `25.6.0`）                                  |

---

## 2. 機能要件（FR）

| ID        | 要件                                                                                                                                           |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| **FR-01** | 接続抽象化：URL／Engine／Session 生成を `DatabaseManager` に統合                                                                                           |
| **FR-02** | 多様な認証：PW／SSL／Kerberos／IAM／OAuth2 などをプラグイン化                                                                                                   |
| **FR-03** | 同期 (`DatabaseManager`) と非同期 (`AsyncDatabaseManager`) API                                                                                     |
| **FR-04** | コネクションプール細分設定                                                                                                                                |
| **FR-05** | トランザクション ContextManager で自動 commit/rollback                                                                                                  |
| **FR-06** | CRUD・生 SQL・メタデータ取得ユーティリティ                                                                                                                    |
| **FR-07** | ORM 連携（Declarative Base 共有）                                                                                                                  |
| **FR-08** | DB 依存例外を独自階層へ正規化                                                                                                                             |
| **FR-09** | 標準 `logging` + OpenTelemetry トレーシング                                                                                                          |
| **FR-10** | Alembic 用設定テンプレート生成                                                                                                                          |
| **FR-11** | `connect_info()`：接続済み Engine から dialect/driver 等を取得                                                                                          |
| **FR-12** | 上記情報をログやエラー表示に再利用                                                                                                                            |
| **FR-13** | `AsyncDatabaseManager.connect_info_async()` も提供                                                                                              |
| **FR-14** | **探索的 URL 生成** `discover_connection()`／`discover_connection_async()`<br> ― 基本情報（user/password/host/port/database）だけで接続テストを順次実施し、成功した URL を返す |
| **FR-15** | `DatabaseManager.from_discovery()` と非同期版で簡易利用                                                                                                |
| **FR-16** | 探索対象 dialect・タイムアウトをユーザー指定可能                                                                                                                 |
| **FR-17** | 上記機能をドキュメントとテストでカバレッジ確保                                                                                                                      |

## 4. 採用 DB ドライバと根拠

| DB              | 同期ドライバ            | 非同期ドライバ        | 採用理由（抜粋）          |
| --------------- | ----------------- | -------------- | ----------------- |
| PostgreSQL      | `psycopg2-binary` | `asyncpg`      | 公式既定・高速・wheel 提供  |
| MySQL / MariaDB | `mysqlclient`     | `aiomysql`     | C 実装高速・採用例多数      |
| SQLite          | 標準 `sqlite3`      | `aiosqlite`    | CPython 同梱・API 互換 |
| Oracle          | `python-oracledb` | 同左（async 対応）   | Oracle 公式後継       |
| SQL Server      | `pyodbc`          | `aioodbc` (任意) | 公式方言前提・全 OS wheel |

## 7. 主要 API シグネチャ

```python
class DatabaseManager:
    def __init__(self, url: str | None = None, *, config: DBConfig | dict | None = None, ...)

    # 自動判定
    def connect_info(self) -> ConnectionInfo: ...

    # 探索的 URL 生成
    @classmethod
    def discover_connection(cls, *, user: str, password: str, host: str = "localhost",
                            port: int | None = None, database: str | None = None,
                            dialects: Sequence[str] = DEFAULT_DIALECTS,
                            timeout: float = 5.0) -> str: ...

    # 簡易構築
    @classmethod
    def from_discovery(cls, **kwargs) -> "DatabaseManager": ...
```

非同期版 (`AsyncDatabaseManager`) は同等の `async`/`await` メソッドを備えます。

## 9. CI/CD 概要

### ci.yml

1. OS×Python マトリクスで **ruff → mypy → pytest(cov) → wheel**
2. タグ push 時に **Poetry publish (PyPI)**

### docs.yml

1. Poetry install (`docs` extra) → `mkdocs build --strict`
2. `actions/upload-pages-artifact` → `actions/deploy-pages`

## 10. 開発ワークフロー
